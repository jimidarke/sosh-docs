    Cases¶

    UAT-CLD-01 — Operator can log into admin console¶

    Field Value
    Tier [CLOUD]
    Category Account & Access
    Pre Tester has UAT username + password.
    Steps 1. Open https://sovereignshelf.net/admin/login
    2. Enter username + password
    3. Click Sign In
    Expected Within 5 seconds you land on the admin dashboard and see your name (or username) on the page, plus a list/summary of stores/tags.

    UAT-CLD-02 — Wrong password is rejected clearly¶

    Field Value
    Tier [CLOUD]
    Category Account & Access
    Pre UAT-CLD-01 passed.
    Steps 1. Log out (top-right menu, or /admin/logout)
    2. Try to log in with username + password wrong
    Expected You stay on the login page. A clear error message tells you the credentials were wrong. The page does not crash, redirect to a stack trace, or hang.

    UAT-CLD-03 — Change-password flow works¶

    Field Value
    Tier [CLOUD]
    Category Account & Access
    Pre Logged in as UAT user.
    Steps 1. Visit /admin/change-password
    2. Enter current password
    3. Enter new password
    4. Confirm and submit
    5. Log out, log back in with the new password
    6. Change it back to the original
    Expected Each step succeeds with a clear confirmation. The new password works on next login. Reverting works the same way.

    UAT-CLD-04 — Logout actually logs you out¶

    Field Value
    Tier [CLOUD]
    Category Account & Access
    Pre Logged in.
    Steps 1. Click Sign Out
    2. With the browser back button, try to return to the admin dashboard
    Expected You are sent back to the login page, not shown cached admin content. The back button does not let you bypass authentication.
