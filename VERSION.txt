## Changelog

### 0.7.0 (2026-07-14)
- Add CI (`.github/workflows/docs.yml`). The public site was published only by a hand-run
  `mkdocs gh-deploy`, so `gh-pages` trailed whatever commit the last manual deploy
  happened to run from, and `strict: true` failures surfaced at deploy time rather than
  when the offending commit landed
- Build `--strict` on every push to `main` and every PR; publish only on a `v*.*.0` release
  tag (major.minor.release, release component zero). Patch tags build but do not deploy
- The deploy job declares `needs: build`, so the strict build is the gate a release must
  clear, not just an advisory check on `main`
- Deploys via `mkdocs gh-deploy`, not `actions/deploy-pages`: Pages is on its legacy branch
  source and the `docs.sovereignshelf.net` certificate is still provisioning, so flipping
  the source risks resetting that request. `docs/CNAME` rides through the build

### 0.6.0 (2026-07-13)
- Fill the shot list: 21 of the 33 open `!!! screenshot` placeholders are now real
  images, wired into their pages. Gaps (a placeholder the walker has no recipe for)
  went 21 → 0 on the browser surfaces
- Add a third shotwalker surface: the staff handheld, over adb (`adb.py`,
  `walk_app.py`). Playwright cannot see an Android app, which is why the eight
  `assets/app/*` shots sat unautomated while both web surfaces were covered. Nodes are
  found by text via `uiautomator dump`, never by coordinate; callouts are drawn onto
  the bitmap since there is no DOM to outline. Six of the eight are captured
- **The app surface is not reveal-only, and cannot be** — its lessons *are* writes
  (saving a multi-product grid, assigning a will-call order). `guard.py` protects the
  browser surfaces and does nothing here, so this half is a bench tool and must never
  be pointed at a store's phone. `willcall-assign-order` releases what it assigns, so
  a run leaves the bench as it found it
- Phone screens get their own redaction pass (`redact.app_masks`): the app prints a
  store UUID, and this repo is public
- New report bucket, **blocked** — a shot no recipe can ever take, with the reason.
  Distinct from a gap on purpose: a gap is a to-do, and a to-do list that can never
  reach zero stops being read. Nine shots are blocked, and none of them is blocked for
  want of effort: four POS `*-test-ok` shots need a Test POST the network guard aborts;
  `pos-localapi-keys` and `pos-odoo-form` are art-directed *at* fields that
  `redact.py` blacks out (an API key; the appliance's Base URL), so they would publish
  a highlighted black rectangle; `image-library-crop` needs a Designer feature that is
  broken; the two wizard shots need an unpaired handheld
- The guard only ever tightened: `#sql/spire/odoo-test-btn` (a Test opens an outbound
  connection to a customer's POS with the credentials in the form — a real side effect
  on someone else's Commander), `.purpose-save`, `#upload-image-btn`, `#sync-now-btn`,
  and the will-call release/delete/assign form submits
- Commander 1.0.0-20260713-175438 fixes the black-on-black Templates dropdowns, so
  `templates-defaults-card` and `templates-purpose-pill` are unheld and published.
  `templates-defaults-pair` stays held — its blocker was never the colour, it asks for
  a "Saved tick" that only exists after a write

  Two defects found while shooting, both worth fixing in `sosh`:
  - `/admin/sales-orders`: the **Sync now button is invisible**. It carries Bootstrap's
    `btn-outline-light` (white on white) on a near-white page. The primary action on
    the page cannot be seen by anyone. `sales-orders-page` is held on it
  - Designer: **the Image Library cannot be opened at all**. `designer/index.html`
    declares `id="library-modal"` twice, so `getElementById` returns the Shared Template
    Library and the Image Library never leaves `display:none` — the feature c06
    documents is unreachable in the UI

### 0.3.0 (2026-07-13)
- Add `tools/shotwalker/`: a Playwright harness that walks the Commander console
  (`:8080`) and the Template Designer (`:8090`), captures the screenshots the docs
  ask for, and smoke-tests every page on the way through
- The docs were already the shot list — every page carried a `!!! screenshot`
  placeholder naming its exact target file. Shotwalker reads those as its work
  list and answers each with a recipe keyed to the same filename, so the two can
  be diffed: a placeholder with no recipe is a gap, a recipe with no placeholder
  is stale. That diff is what keeps it honest as the UI drifts
- Wire in 25 of the 29 browser-capturable shots; 4 stay placeholders on purpose
  (see below). Console tables render client-side from `/api/*`, so a real browser
  is the only thing that can see them — `curl` gets an empty shell
- Reveal-only, enforced in code, not by discipline: a network guard aborts every
  non-GET request outside a narrow allowlist, and a DOM guard swallows clicks on
  mutating controls. Safe to point at any Commander, including a customer's
- Secrets are masked at capture time (PIN, API keys, POS passwords), so they never
  land in a PNG — these images are committed to a public repo
- Smoke sweep: 23/23 pages healthy. Two findings worth knowing — the console draws
  a broken `<img>` for tags with no pushed image (`/tag-image/<addr>` 404s; real
  device state, 14 on `/fleet`), and `/admin/mobile-binding` mints its pairing QR
  with a POST *on page load*
- Held back, deliberately: `templates-defaults-card`, `templates-defaults-pair`,
  and `templates-purpose-pill` — the Templates-page dropdowns render black-on-black
  in `sosh`, and the dropdown is the subject of all three. A known-bad image in
  permanent docs is worse than a placeholder that admits the gap.
  `templates-defaults-pair` has a second blocker: it asks for the "Saved tick",
  which only appears after a write, so it is not capturable reveal-only as specced
- `templates-library-card` also stays open: the bench Commander's Template Library
  is empty, so the card reads "No library templates available" while the lesson
  describes a Download button. Seed the library, then `--only` it
- Re-file three Templates-page shots from `assets/designer/` to `assets/console/` —
  they are console screenshots; `c12` says so in its own text
